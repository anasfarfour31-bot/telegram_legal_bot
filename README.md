# بوت منشورات قانونية لقناة تيليجرام عبر GitHub Actions

هذا المشروع يرسل **3 منشورات يوميًا** إلى قناة تيليجرام في تشغيل واحد تلقائيًا عند الساعة **08:00 صباحًا بتوقيت إسطنبول/تركيا**.

لأن GitHub Actions يستخدم توقيت UTC، فالجدولة مضبوطة على:

```yaml
- cron: "0 5 * * *"
```

وهذا يساوي الساعة **08:00 صباحًا في تركيا** لأن توقيت تركيا UTC+3.

المنشورات موجودة في:

```text
data/posts.json
```

وسجل المنشورات المرسلة موجود في:

```text
data/state.json
```

البوت لا يكرر المنشورات حتى تنتهي القائمة، ثم يبدأ دورة جديدة تلقائيًا.

## 1) إنشاء بوت تيليجرام

1. افتح تيليجرام وابحث عن `@BotFather`.
2. أرسل الأمر `/newbot`.
3. اختر اسمًا للبوت واسم مستخدم ينتهي بـ `bot`.
4. انسخ التوكن الذي يعطيك إياه BotFather.
5. أضف البوت إلى قناتك كـ Admin، وأعطه صلاحية النشر.

## 2) رفع المشروع إلى GitHub

1. أنشئ Repository جديد في GitHub.
2. ارفع كل ملفات هذا المشروع كما هي.
3. تأكد أن ملف workflow موجود في:

```text
.github/workflows/telegram-posts.yml
```

## 3) إضافة الأسرار في GitHub

اذهب إلى:

```text
Repository Settings > Secrets and variables > Actions > New repository secret
```

أضف السرّين التاليين:

```text
TELEGRAM_BOT_TOKEN
```

ضع داخله توكن البوت.

```text
TELEGRAM_CHAT_ID
```

ضع داخله معرف القناة. إذا كانت القناة عامة، يمكنك استخدام اسمها مثل:

```text
@your_channel_username
```

إذا كانت القناة خاصة، استخدم الرقم الداخلي للقناة بعد استخراجه بالطريقة المناسبة.

## 4) اختبار الإرسال يدويًا

اذهب إلى تبويب:

```text
Actions > Send Telegram Legal Posts > Run workflow
```

ثم شغّل workflow يدويًا. إذا نجح، ستصل **3 منشورات** إلى القناة.

## 5) تعديل عدد المنشورات اليومية

العدد مضبوط داخل ملف:

```text
.github/workflows/telegram-posts.yml
```

في هذا الجزء:

```yaml
POSTS_PER_RUN: "3"
```

لتغيير العدد إلى 5 مثلًا:

```yaml
POSTS_PER_RUN: "5"
```

## 6) تعديل وقت النشر

الموعد موجود في ملف:

```text
.github/workflows/telegram-posts.yml
```

السطر الحالي:

```yaml
- cron: "0 5 * * *"
```

معناه: يوميًا الساعة 05:00 UTC = الساعة 08:00 صباحًا في إسطنبول.

أمثلة:

```yaml
# يوميًا الساعة 09:00 صباحًا في تركيا
- cron: "0 6 * * *"

# يوميًا الساعة 10:30 صباحًا في تركيا
- cron: "30 7 * * *"
```

## 7) إضافة توقيع ثابت أسفل كل منشور

من GitHub اذهب إلى:

```text
Settings > Secrets and variables > Actions > Variables > New repository variable
```

أضف متغيرًا باسم:

```text
POST_FOOTER
```

مثال للقيمة:

```text
للاستشارة القانونية تواصل معنا.
```

اتركه فارغًا إذا لا تريد توقيعًا.

## ملاحظات مهمة

- لا تضع توكن البوت داخل الملفات أبدًا.
- اجعل المستودع Private إذا كانت المنشورات خاصة أو لا تريد نشر بنك المحتوى للناس.
- GitHub Actions قد لا ينفذ cron في الدقيقة نفسها دائمًا، وهذا طبيعي.
- يوجد تأخير 5 ثوانٍ بين المنشورات الثلاثة لتجنب إرسالها دفعة واحدة بسرعة كبيرة.
- إذا فشل الإرسال، افتح تبويب Actions واقرأ رسالة الخطأ.
